ModulePal

Web app enabling Warwick students to review academic modules, helping them make better decisions.
  • Verifies a Warwick student's enrolment status via OAuth, and only allows them to make reviews if they took a module to ensure legitimacy.
  • Harnesses Warwick student data via OAuth, personalising the app to their university account.
  • Anonymously contextualises reviews based on the grade students achieved in the module.
  • Used semi-offically in Warwick's Computer Science department, with >1500 reviews from ~100 users.
  • Secure Spring (Java) backend, using an in-memory ORM (SQL/Hibernate) database for caching, and a NoSQL Firebase Realtime Database for persistant data storage. Provides a REST API for the frontend.
  • React (TypeScript) frontend utilising hooks and other advanced features, harnessing Firebase Authentication to manage authentication states.
  • Backend packaged as a WAR application and hosted on a Tomcat instance on a DigitalOcean Droplet, frontend hosted via Firebase Hosting.
  • Highlly optimised with an efficient NoSQL data schema, scalable metric computations via moving averages, and paginated queries.

    • TypeScript   Java   React   Spring

    OAuth1-HMAC

    Lightweight library implementing the OAuth1.0 protocol as a consumer, using the HMAC signing strategy.
  • Implements the OAuth1.0 flow for a generic consumer via Google's OAuth Client Library for Java.
  • Provides an easy-to-use external interface for generic OAuth1.0 consumers.
  • Fully documented with a JavaDoc.
  • Listed as an official OAuth1.0 library on oauth.net/1.
  • Provided an example implementation via Spring for Warwick University's OAuth1.0 provider, which is officially listed on their website.
  • Used in my ModulePal app for the Warwick OAuth1.0 flow.

    • Java
